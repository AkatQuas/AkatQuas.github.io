设计一个 CLI 工具

June 11, 2022 · 7 min read

自从接手编辑器的 CLI 工具以来，我对原始的代码设计深恶痛绝。只是无奈重构、升级等迟迟无法提上日程，所以只能在这里做大梦。

常见工具​

在 Node 中，我接触过的 CLI 开发工具有

  • common-bin
  • commander
  • oclif

相比较而言，前两个 common-bin 和 commander 只能算是 CLI 的开发工具套件 (library) ，而 oclif 可以称得上 CLI 开发框架。

用 Web 生态去类比，前两者好比就是 React 和 Vue ，属于 Web UI library 。作为核心库，它们只提供了 MVVM 的功能，诸如 router 、store 、http 等都需要同生态下的其他 library 支持。

It's recommended to go through the section Item 13: Know the Differences Between type and interface in the book Effective TypeScript written by Dan Vanderkam.

In most cases, they are interchangeable when doing type gymnastics.

interface IA = {};

type TB = TA & {};

重构 Electron 应用之后的感悟

March 25, 2022 · 4 min read

自从转岗以来，基于 Electron 开发编辑器应用已经快半年了，主要载体是 VS Code 。

现行的架构主要是利用 VS Code 提供的插件系统，注入相应的业务实现。该方案有很大的性能问题，是业务组件只能 VS Code 编辑器启动之后，才能初始化、实例化，性能有很大影响。另外，鉴于业务实现都跑在 VSC 的插件进程系统，通信显然会成为问题。

当业务复杂度上升之后，当前的设计方案会成为一个瓶颈。

这两个月刚好赶上了重构，主要思路就是将组件模块化， VSC 变成应用的一个模块，亦即编辑区，原先的插件式组件，升级成为与 VSC 同等级的模块。有这样的设计，应用的横向扩展性、模块间的联系性都能得到很大的提升。

Class Asynchronous Initialization

July 13, 2021 · 2 min read

Sometimes, the instance of Class is not completely initialized yet when we first see it, because its property is initialized asynchronously.

class A {
  data: string;
  constructor() {
    // asynchronous data
    Promise.resolve('data').then(d => void this.data = d);
  }
}

const a = new A();
assert.equal(a.data, undefined);
setTimeout(() => {
  assert.equal(a.data, 'data');
}, 0)
