业务需要支持多重条件的筛选，最好的工具就是 query builder，下面从底向上地给出一个简单的 QueryBuilder 的设计。

先从条件 query 开始考虑，这个条件是 atomic 的记做 Q。我们需要：

  1. 一个 key 字段，用来表示 query 的索引域，（可以简单地认为是字段名称，但是 mongo 里面用的是路径）。

  2. 一个 operator 字段，表示 这个索引跟后面的值直接用什么操作符，数值/字符串/正则 等应当有所不同。

  3. 一个 value 字段，表示前面 索引域 的界定限度。

Redux 源码读后感

February 10, 2020 · 16 min read

redux version is 4.0.5.

redux 本身的设计很简单，主要是通过 pure function 来改变 store 里面 state 的值。

source code

// 每一次在组件层面或者 store 直接调用 `dispatch`
// 经过相关中间件的执行之后，最后的函数会到这里
function dispatch(action) {
  // 一些错误判断，预防机制
  if (!isPlainObject(action)) {
    throw new Error(
      'Actions must be plain objects. ' +
        'Use custom middleware for async actions.'
    );
  }

  if (typeof action.type === 'undefined') {
    throw new Error(
      'Actions may not have an undefined "type" property. ' +
        'Have you misspelled a constant?'
    );
  }

  if (isDispatching) {
    throw new Error('Reducers may not dispatch actions.');
  }

  try {
    isDispatching = true;
    // pure function 更新 state
    currentState = currentReducer(currentState, action);
  } finally {
    isDispatching = false;
  }

  // subscribed 的回调在这里触发
  // ...

  return action;
}

Zero-width space

November 1, 2019 · One min read

Insert zero-width space to cheat keywords detection without losing reading convenience.

function insertSpaces(string) {
  return Array.from(string).join('\u200b');
}

In Unicode, the zero-width space character is encoded as U+200B (HTML entity &#8203;)
