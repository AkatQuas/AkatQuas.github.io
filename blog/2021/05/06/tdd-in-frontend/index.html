Test-Driven Development in FrontEnd

May 6, 2021 Â· 2 min read

A quick summary on how to write tests in frontend framework. Using ReactJS as example codes, but these principles are also suitable for other frameworks such as Vue.js or Angular.

How to test componentsâ€‹

mount / render without problem

it('renders without crashing', () => {
  const div = document.createElement('div');
  ReactDOM.render(<App />, div);
  ReactDOM.unmountComponentAtNode(div);
});

expect some specific html tag or text content to be correct

it('renders the app and the heading', () => {
  const wrapper = mount(<App />);
  expect(wrapper.find('h1').text()).toBe('Hello React');
  expect(wrapper.find('.counter label').text()).toBe('Current');
  expect(wrapper.find('.counter span').text()).toBe('0');
});

expect user event works well, and mutate the state correctly

it('should increment the count by one', () => {
  const wrapper = shallow(<Counter />);
  expect(wrapper.find('.counter span').text()).toBe('1');
  wrapper.find('.counter').simulate('click');
  expect(wrapper.find('.counter span').text()).toBe('2');

  wrapper.find('.counter').simulate('click', { shiftKey: false });
  expect(wrapper.find('.counter span').text()).toBe('3');

  wrapper.find('.counter').simulate('click', { shiftKey: true });
  expect(wrapper.find('.counter span').text()).toBe('12');
});

expect some callback to be called correctly

it('should call the handler on click', () => {
  const handler = jest.fn();
  const wrapper = shallow(<Counter count={1} onCounterIncrease={handler} />);
  wrapper.find('.counter').simulate('click', { shiftKey: false });
  expect(handler).toBeCalledWith({ shiftKey: false });
});

it('should call the instance method correctly', () => {
  const wrapper = shallow(<App />);
  const instance = wrapper.instance();
  expect(instance.state.count).toBe(0);
  instance.increment(false);
  expect(instance.state.count).toBe(1);

  instance.increment(true);
  expect(instance.state.count).toBe(11);
});

expect some time-relevant callback take side effect correctly

jest.useFakeTimers();

test('waits 1 second before ending the game', () => {
  // some rendering and user event action
  expect(setTimeout).toHaveBeenCalledTimes(1);
  expect(setTimeout).toHaveBeenLastCalledWith(expect.any(Function), 1000);
});

expect side effect works correctly

this can also be tested in e2e testing such as Cypress.

// react testing library
it('should change the document title correctly', () => {
  act(() => {
    ReactDOM.render(<Counter />, container);
  });
  const button = container.querySelector('button');
  expect(document.title).toBe('You clicked 0 times');

  act(() => {
    button.dispatchEvent(new MouseEvent('click', { bubbles: true }));
  });

  expect(document.title).toBe('You clicked 1 times');
});
