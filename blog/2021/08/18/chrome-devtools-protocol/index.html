Fun with Chrome Devtools Protocol

August 18, 2021 ¬∑ 5 min read

The Chrome DevTools Protocol allows for tools to instrument, inspect, debug and profile Chromium, Chrome and other Blink-based browsers.

The most familiar Chrome DevTools uses this protocol to help debugging webpages.

Under the hood, the protocol relies on WebSocket to work, wiki. The commands and events are serialized JSON objects on the protocol.

The package chrome-remote-interface is a good tool, and we are going to demonstrate some cases with it.

Debugging a node process‚Äã

First let's start a node process with inspect option.

$ node --inspect=9222 server.js

# output
Debugger listening on ws://127.0.0.1:9222/d4231410-13e5-463d-a5ba-23d206432a35
#                       ^ debugger endpoint

Example express server.
const express = require('express');
const app = express();
const port = 3000;

const marquee = ['log', 'warn', 'debug', 'error'];

let id = 0;
function nonce() {
  id = (id + 1) % 4;
  const method = marquee[id];
  console[method](`${method} works`);
}

app.get('/', (req, res) => {
  nonce();
  res.send('Hello World!');
});

app.listen(port, () => {
  console.log(`Example app listening at http://localhost:${port}`);
});

With chrome-remote-interface, it's very easy to connect to the debugger endpoint.

const CDP = require('chrome-remote-interface');

const client = await CDP({
  target: 'ws://127.0.0.1:9222/d4231410-13e5-463d-a5ba-23d206432a35',
});
const { Debugger, Console, Runtime } = client;
client.on('Runtime.consoleAPICalled', (params) => {
  console.log('received: ', params);
});
await Runtime.runIfWaitingForDebugger();
await Runtime.enable();
await Debugger.enable({
  maxScriptsCacheSize: 10000000,
});
await Console.enable();

Debugging a webpage‚Äã

This is much easier to debug a webpage since we could use the already powerful Devtools built with Chrome.

The quick way is to open chrome://inspect/#pages and you can start debugging.

But how to debug the webpage using chrome-remote-interface ?

First, we need some webpage running with available debugging port.

In this case, we'll start with the electron quick-start project.

After cloning and installing the dependencies, add a line before app.whenReady() in main.js to enable remote debugging.

+ app.commandLine.appendSwitch('remote-debugging-port', '9222');

// This method will be called when Electron has finished
// initialization and is ready to create browser windows.
// Some APIs can only be used after this event occurs.
app.whenReady().then(() => {

So the time we start the application using npx electron .Ôºå it would output something like this

DevTools listening on ws://127.0.0.1:9222/devtools/browser/a1df1a72-9dce-42ad-80ed-d8765dbb7520
#                       ^ devtools debug endpoint

However, this websocket url is only a devtools debug endpoint, not the chrome devtool protocol debug endpoint. We have to visit the http://localhost:9222/json to find out the real webSocketDebuggerUrl for chrome-remote-interface to connect.

Example output 
[
  {
    "description": "",
    "devtoolsFrontendUrl": "/devtools/inspector.html?ws=localhost:9222/devtools/page/D6F9769D8A20DCF2E8F574D231F9A8C6",
    "id": "D6F9769D8A20DCF2E8F574D231F9A8C6",
    "title": "Hello Vanilla!",
    "type": "page",
    "url": "file:///Users/workspace/quick-start/index.html",
    "webSocketDebuggerUrl": "ws://localhost:9222/devtools/page/D6F9769D8A20DCF2E8F574D231F9A8C6"
    //    ^ debugger endpoint
  }
]

With the webSocketDebuggerUrl, we can write some code to interact with the webpage.

const CDP = require('chrome-remote-interface');

const options = {
  target: 'ws://localhost:9222/devtools/page/D6F9769D8A20DCF2E8F574D231F9A8C6',
};

CDP(options, async (client) => {
  console.log('Connected!');
  const { Page } = client;
  await Page.enable();

  await Page.navigate({ url: 'https://cn.bing.com' });
  await Page.loadEventFired();
  client.close();
}).on('error', (err) => {
  console.error(err);
});

You'll find out the page is navigated if you check out the electron window.

We have more abilities to manipulate the webpage. Just take a look at the listed protocol.

Tricky‚Äã

Look at the output json, there's a property named devtoolsFrontendUrl.

We would get the devtool panel if visiting http://localhost:9222/<devtoolsFrontendUrl>. Then open the devtool for the devtool page, for example, press F12 or cmd+option+I.

We will find the message communication in devtool panel Network>WS.

These are the commands sent via chrome devtools protocol, we could do the same thing using chrome-remote-interface.
