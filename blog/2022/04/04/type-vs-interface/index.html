Type vs Interface

April 4, 2022 Â· 2 min read

It's recommended to go through the section Item 13: Know the Differences Between type and interface in the book Effective TypeScript written by Dan Vanderkam.

In most cases, they are interchangeable when doing type gymnastics.

type TA = { };
interface IA = {};

type TB = TA & {};
interface IB extends IA {};

class A implements TA {};
class AA implements IA {};

However, there are some features you can only express with type, not interface.

Union

There's no syntax available when writing unions with interface.

type TC = TA | IA | {};
// oops, there's no union of interfaces.

Tuple

You can't create a tuple directly from interface.

type Pair = [number, string];
type StringList = string[];
// oops, there's no tuple directly from interface

However, here's a trick way to express tuple in interface

interface ITuple {
  0: number;
  1: string;
  length: 2;
}

const x: ITuple = [42, '42'];

Mapped types

It's invalid to distribute types in interface.

type Fruit = 'apple' | 'orange' | 'banana';

// this is ok in type
type FruitCount = {
  [key in Fruit]: number;
};

// oops, this is invalid
interface FruitCount {
  // error, ts(7061)
  [key in Fruit]: number;
}

In general, it's more safe and robust to use type.

I set up consistent-type-definitions rule in my eslint config.

@typescript-eslint/consistent-type-definitions: [ 'error', 'type']

But, interface does have one ability that type doesn't.

I haven't find more, maybe add more in the future.

Declaration merging (augment)

interface IA {
  name: string;
}
interface IA {
  age: number;
}

const a: IA = { name: 'n', age: 2 };

There's one blog provided by TypeScript team that talks about the preferences when considering compile performance. Take a look if you're interested.

In conclusion, You'd better know the differences between interface and type, and make the decision that's suitable for your project.
