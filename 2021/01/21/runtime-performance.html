Frame end

You won’t always necessarily touch every part of the pipeline on every frame.

Each of these parts of the pipeline represents an opportunity to introduce jank.

Analyzing Runtime Performance

Tools to analyze runtime performance

  • Don’t write JavaScript that forces the browser to recalculate layout. Separate read and write functions, and perform reads first.
  • Don’t over-complicate your CSS. Use less CSS and keep your CSS selectors simple.
  • Avoid layout as much as possible. Choose CSS that doesn’t trigger layout at all.
  • Painting can take up more time than any other rendering activity. Watch out for paint bottlenecks.

Optiziming Rendering Performance

  • optimize JavaScript execution

    • Avoid using setTimeout or setInterval for visual updates; use requestAnimationFrame instead.
    • Move long-running JavaScript off the main thread to Web Workers.
    • Use micro-tasks to make DOM changes over several frames.
    • Leveraging requestIdleCallback if you can.

  • Reduce the Scope and Complexity of Style Calculations

    • Reduce the complexity of your selectors; use a class-centric methodology like Block-Element-Modifier.
    • Reduce the number of elements on which style calculation must be calculated.

  • Avoid Large, Complex Layouts and Layout Thrashing

    • Layout is normally scoped to the whole document.
    • The number of DOM elements will affect performance; you should avoid triggering layout wherever possible.
    • Assess layout model performance; new Flexbox is typically faster than older Flexbox or float-based layout models.
    • Avoid forced synchronous layouts and layout thrashing; read style values first and then make style changes.

  • Simplify Paint Complexity and Reduce Paint Areas

    • Changing any property apart from transforms or opacity always triggers paint.
    • Paint is often the most expensive part of the pixel pipeline; avoid it where you can.
    • Reduce paint areas through layer promotion and orchestration of animations.

  • Stick to Compositor-Only Properties and Manage Layer Count

    • Stick to transform and opacity changes for your animations.
    • Promote moving elements with will-change or translateZ.
    • Avoid overusing promotion rules; layers require memory and management.

  • Debounce the Input Handlers

    • Avoid long-running input handlers; they can block scrolling.
    • Do not make style changes in input handlers, otherwise it will impact requestAnimationFrame.
    • Debounce the handlers; store event values and deal with style changes in the next requestAnimationFrame callback.
